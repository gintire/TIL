# Commands and Arguments
Docker 명령어에서 argument를 전달해주는 것을 manifest 작성에 적용해본다.

dockerfile
```
FROM Ubuntu
ENTRYPOINT ["sleep"]
CMD ["5"]
```

###도커 명령어를 manifest로 변경
docker command
```
docker run --name ubuntu-sleeper \
    ubuntu-sleeper 10
```

pod-definition.yaml
```
apiVersion: v1
kind: Pod
metadata:
  name: ubuntu-sleeper-pod
spec:
  containers:
    - name: ubuntu-sleeper
      image: ubuntu-sleeper
      args: ["10"]
```

```
kubectl create -f pod-definition.yaml
```

### Entrypoint override를 원할 경우  
docker
```
docker run --name ubuntu-sleeper \
    --entrypoint sleep2.0
    ubuntu-sleeper 10
```

pod-definition.yaml
```
apiVersion: v1
kind: Pod
metadata:
  name: ubuntu-sleeper-pod
spec:
  containers:
    - name: ubuntu-sleeper
      image: ubuntu-sleeper
      command: ["sleep2.0"]
      args: ["10"]
```
